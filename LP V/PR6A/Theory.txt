Aim
---

To implement the Bully Algorithm for leader election in a distributed system to ensure that a coordinator is always available to manage system activities\[cite: 10\].

Theory
------

In a distributed system, a leader or coordinator is often required to perform specific centralized tasks. However, if this coordinator fails, the system must have a way to elect a new one without human intervention. The Bully Algorithm is one of the most common methods for this. It assumes that every process in the system has a unique, numerical ID\[cite: 10\]. The core philosophy of the algorithm is simple: the process with the highest ID that is currently active is "bullied" into being the leader by the other processes, or it asserts its dominance over those with lower IDs\[cite: 10\].

The election process is triggered when any process notices that the current coordinator is no longer responding. That process sends an "Election" message to all processes with an ID higher than its own\[cite: 10\]. If any of those higher-ID processes are alive, they respond with an "Alive" (or "OK") message, effectively telling the initiator, "I'll take it from here"\[cite: 10\]. The higher-ID process then starts its own election by messaging even higher-ID processes. If no higher-ID processes respond, the current process knows it is the highest active node in the network\[cite: 10\].

Once a process determines it is the winner, it sends a "Coordinator" message to all other processes in the system to announce its new role\[cite: 10\]. A unique feature of this algorithm is that if a process with a very high ID (like the original coordinator) crashes and then recovers, it immediately restarts the election process\[cite: 10\]. Because it has a higher ID than the current leader, it will "bully" the current leader out of its position and regain control\[cite: 10\]. This ensures the system always converges toward the most capable (highest-ID) available node.

Explanation of Logic:  
The algorithm works by having a process check its "superiors" (processes with higher IDs)\[cite: 10\]. If an "Alive" message comes back from a higher process, the lower process steps back and waits\[cite: 10\]. The logic uses a simple loop to check the state array to see which processes are currently functioning\[cite: 10\]. The final coordinator is always the one with the highest index in that array whose state is true\[cite: 10\].

The implementation of the Bully Algorithm shows effective mechanism for dynamic leader election in distributed environments\[cite: 10\]. Any process can initiate an election, the program automatically promotes the highest available node to the coordinator role.
