Aim

To implement the Ring Algorithm for leader election in a distributed system, ensuring that a new coordinator is chosen following the failure of the existing leader\[cite: 11\].

Theory
------

The Ring Algorithm is a decentralized leader election method used in distributed systems where all processes are organized in a logical circle. Unlike the Bully algorithm, which relies on a hierarchy of "bullying" lower processes, the Ring algorithm relies on a sequential flow of information. Each process in the ring is aware of its neighbors and knows the overall structure of the network. When a process notices that the current coordinator has failed, it initiates an election by creating an "Election Message"\[cite: 11\].

The election process occurs in a circular fashion. The initiator sends the election message to its immediate successor in the ring. If the successor is down, the initiator skips it and sends the message to the next active process. As the message moves around the ring, each active process adds its own unique identifier (ID) to the message or compares its ID to the current maximum found in the message\[cite: 11\]. This continues until the message returns to the original initiator, completing a full circle. By the time the message returns, the initiator can identify which active process has the highest ID\[cite: 11\].

Once the process with the highest ID is identified as the new coordinator, a second message is circulated around the ring. This "Coordinator Message" informs every active node of the identity of the new leader\[cite: 11\]. This ensures that every process in the distributed system is synchronized and knows which node to contact for centralized tasks. The algorithm is efficient in terms of message overhead and ensures that a leader is always elected as long as at least one process remains active in the ring\[cite: 11\].
